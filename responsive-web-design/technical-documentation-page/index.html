Introduction

This document discusses the calculation of the six trigonometric functions in Python 3. The accompanying code samples are designed to run in Python 3.7 or greater.

For a mathematical explanation of the functions, see Wikipedia's entry.

Sine

In mathematics, the sine of the variable x would be written as sin(x). It is written the same whether x is expressed in degrees or in radians.

In Python, the function can be imported from the math package (which just uses the underlying C implementation) and is similarly named sin, but the function argument must always be expressed in radians. To use degrees you can first convert the degree measure into radians using the math package function radians. The following example computes and displays various sine values, and includes some degrees to radians conversions.

Input:

from math import sin, radians x = 0.5 print(f'sin({x}) ≈ {sin(x)}') for degree_measure in [0, 30, 45, 60, 90]: radian_measure = radians(degree_measure) print(f'sin({degree_measure:>2}°) ' f'≈ sin({radian_measure:.8f}) ' f'≈ {sin(radian_measure):.8f}')

Output: sin(0.5) ≈ 0.479425538604203 sin( 0°) ≈ sin(0.00000000) ≈ 0.00000000 sin(30°) ≈ sin(0.52359878) ≈ 0.50000000 sin(45°) ≈ sin(0.78539816) ≈ 0.70710678 sin(60°) ≈ sin(1.04719755) ≈ 0.86602540 sin(90°) ≈ sin(1.57079633) ≈ 1.00000000

  • You can also import the entire math library with the statement import math and then call the same functions with math.sin and math.radians.

Tangent

In mathematics, the tangent of the variable x would be written as tan(x). It is written the same whether x is expressed in degrees or in radians.

In Python, the function can be imported from the math package (which just uses the underlying C implementation) and is similarly named tan, but the function argument must always be expressed in radians. To use degrees you can first convert the degree measure into radians using the math package function radians. The following example computes and displays various tangent values, and includes some degrees to radians conversions.

Input:

from math import tan, radians x = 0.5 print(f'tan({x}) ≈ {tan(x)}') for degree_measure in [0, 30, 45, 60, 90]: radian_measure = radians(degree_measure) print(f'tan({degree_measure:>2}°) ' f'≈ tan({radian_measure:.8f}) ' f'≈ {tan(radian_measure):.8f}')

Output: tan(0.5) ≈ 0.5463024898437905 tan( 0°) ≈ tan(0.00000000) ≈ 0.00000000 tan(30°) ≈ tan(0.52359878) ≈ 0.57735027 tan(45°) ≈ tan(0.78539816) ≈ 1.00000000 tan(60°) ≈ tan(1.04719755) ≈ 1.73205081 tan(90°) ≈ tan(1.57079633) ≈ 16331239353195370.00000000

Unlike sine and cosine, the values of the tangent function are not bounded. You can see the unintended effect of this in the last line of output above. (Also, mathematically tan(90°) is undefined, but using a radian approximation for the angle we see instead that tan(1.57079633) is really large.)

  • You can also import the entire math library with the statement import math and then call the same functions with math.tan and math.radians.
Secant

In mathematics, the secant of the variable x would be written as sec(x). It is written the same whether x is expressed in degrees or in radians. Mathematically, the secant is the reciprocal of the cosine.

In Python, the function does not exist, but we can define such a function as the reciprocal of the cosine function, whose argument must always be expressed in radians. To use degrees you can first convert the degree measure into radians using the math package function radians. The following example defines a sec function and then computes and displays various secant values, including some degrees to radians conversions.

Input:

from math import cos, radians def sec(x): return 1 / cos(x) x = 0.5 print(f'sec({x}) ≈ {sec(x)}') for degree_measure in [0, 30, 45, 60, 90]: radian_measure = radians(degree_measure) print(f'sec({degree_measure:>2}°) ' f'≈ sec({radian_measure:.8f}) ' f'≈ {sec(radian_measure):.8f}')

Output: sec(0.5) ≈ 1.139493927324549 sec( 0°) ≈ sec(0.00000000) ≈ 1.00000000 sec(30°) ≈ sec(0.52359878) ≈ 1.15470054 sec(45°) ≈ sec(0.78539816) ≈ 1.41421356 sec(60°) ≈ sec(1.04719755) ≈ 2.00000000 sec(90°) ≈ sec(1.57079633) ≈ 16331239353195370.00000000

Unlike sine and cosine, the values of the secant function are not bounded (except they cannot be between −1 and 1). You can see the unintended effect of this in the last line of output above. (Also, mathematically sec(90°) is undefined, but using a radian approximation for the angle we see instead that sec(1.57079633) is really large.)

  • You can also import the entire math library with the statement import math and then call the same functions with math.cos and math.radians.
Cosecant

In mathematics, the cosecant of the variable x would be written as csc(x). It is written the same whether x is expressed in degrees or in radians. Mathematically, the cosecant is the reciprocal of the sine.

In Python, the function does not exist, but we can define such a function as the reciprocal of the sine function, whose argument must always be expressed in radians. (To use degrees you can first convert the degree measure into radians using the math package function radians.) There is a bit of an extra complication due to the fact that the rational argument of 0 yields an undefined value. To handle this, we can use the math package floating-point "not a number" (NaN) value nan.

The following example defines a csc function and then computes and displays various cosecant values, including some degrees to radians conversions.

Input:

from math import sin, radians, nan def csc(x): if x == 0: return nan return 1 / sin(x) x = 0.5 print(f'csc({x}) ≈ {csc(x)}') for degree_measure in [0, 30, 45, 60, 90]: radian_measure = radians(degree_measure) print(f'csc({degree_measure:>2}°) ' f'≈ csc({radian_measure:.8f}) ' f'≈ {csc(radian_measure):.8f}')

Output: csc(0.5) ≈ 2.085829642933488 csc( 0°) ≈ csc(0.00000000) ≈ nan csc(30°) ≈ csc(0.52359878) ≈ 2.00000000 csc(45°) ≈ csc(0.78539816) ≈ 1.41421356 csc(60°) ≈ csc(1.04719755) ≈ 1.15470054 csc(90°) ≈ csc(1.57079633) ≈ 1.00000000

Unlike sine and cosine, the values of the cosecant function are not bounded (except they cannot be between −1 and 1). Also, note the "not a number" nan value printed for csc(0). It might be best to test for such a value before continuing with a calculation or output.

  • You can also import the entire math library with the statement import math and then call the same functions with math.sin and math.radians. Also, rather than using math.nan, one could instead use float('NaN') to achieve the same result.
